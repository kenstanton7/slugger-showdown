# ⚾ The Big Fly: Slugger Showdown

**The Big Fly: Slugger Showdown** is a fast-paced, highly polished baseball trivia web game. Players test their historical MLB knowledge by guessing which of two randomly selected players has the higher lifetime career stats.

🎯 **Play the live version here:** https://kenstanton7.github.io/slugger-showdown/ 

---

## 🎮 Game Modes & Dynamic Themes

The game features three distinct modes. Switching modes dynamically alters the UI theme, the player pool, and the in-game audio cues to match the vibe of the stat being tested.

* 💥 **The Big Fly (Home Run Mode)** * **Theme:** Classic MLB Blue & Red
  * **Pool:** Power hitters active between 2000–2026 with **50+ Career Home Runs**.
  * **Audio:** A satisfying *bat crack* for a correct guess, and a *catcher's pop* for a strike.
* 🏏 **The Hit King (Contact Mode)** * **Theme:** Vintage Ballpark Green & Gold
  * **Pool:** Elite contact hitters and veterans active between 2000–2026 with **300+ Career Hits**.
  * **Audio:** *Bat crack* for correct, *catcher's pop* for a strike.
* ⚾ **The Punchout (Strikeout Mode)** * **Theme:** Midnight Slate & Electric Cyan
  * **Pool:** Dominant pitchers active between 2000–2026 with **500+ Career Strikeouts**.
  * **Audio:** *Swapped!* A 99mph *catcher's pop* for a correct guess, and a punishing *bat crack* for a strike.

## ✨ Key Features

* **"On Fire" Streak Tracking:** Rattle off 5 consecutive correct answers without a strike to ignite the UI with a glowing fire animation and trigger a stadium crowd roar.
* **Persistent High Scores:** The game uses local browser storage to save your best score for each specific mode. 
* **Record-Breaking Celebrations:** Beating your high score triggers a hidden "🌟 NEW HIGH SCORE! 🌟" pulsing badge and a special stadium cheer audio cue.
* **Office-Friendly Audio:** Full sound effects suite with a built-in, dynamic 🔊/🔇 Mute toggle.
* **Social Sharing:** Players can instantly share their scores to X (Twitter), WhatsApp, Instagram, or their clipboard directly from the Game Over screen.

## 🛠️ Behind the Code (Engineering Highlights)

This project utilizes a highly robust, custom-built game engine to ensure a seamless user experience without relying on expensive paid APIs.

### 1. The 4-Tier "Ghostbuster" Image Engine
Older, pre-digital era players often lack official modern headshots on MLB's servers. Instead of returning a 404 error, MLB's CDN forcefully returns a generic "grey silhouette" placeholder. To prevent these "ghosts" from ruining the UI, the JavaScript implements a highly advanced validation sequence:
1. **The Digital Scale Check:** The engine invisibly fetches the MLB CDN image and measures the byte size of the returned BLOB. If the image weighs under 12KB (identifying it as a flat, fake silhouette), it intercepts and blocks it.
2. **Wikipedia API Backup:** If MLB fails the scale check, the engine queries the open-source **Wikipedia Action API** for the player's exact name and pulls their primary Wikipedia article photo.
3. **Cache-Busting:** Appends dynamic timestamps (`?v=...`) to image requests to ensure browsers never aggressively cache and serve old placeholders.
4. **Dynamic Initials Graphic:** If a player has no MLB photo *and* no Wikipedia photo, the UI elegantly generates a custom "Trading Card" graphic using the player's initials (e.g., "DD" for David Dellucci), perfectly color-matched to the active game mode.

### 2. Custom Python Data Generation
The player data (`player_pool.js`) is not hardcoded. It is generated by custom Python scraper scripts (`get_pitchers.py`, etc.) that recursively ping the official MLB Stats API. The scripts analyze thousands of active and retired players from 2000–2026, mathematically filter them by specific thresholds (e.g., 500+ Strikeouts), and format them into lightweight JSON arrays for the frontend engine.

## 💻 Tech Stack
* **Frontend:** HTML5, CSS3 (Custom Variables, Keyframe Animations), Vanilla JavaScript (ES6+).

## 🚀 How to Run Locally
1. Clone the repository: `git clone https://github.com/[Your-Username]/[Your-Repo-Name].git`
2. Ensure the audio files (`crack.mp3`, `pop.mp3`, `roar.mp3`, `cheer.mp3`) are in the root directory alongside `index.html`.
3. Open `index.html` in your favorite web browser. No local server setup is required!
